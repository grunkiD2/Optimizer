# Optimizer.Mobile

Native iOS/Android companion app for Optimizer desktop, built with .NET MAUI.

## Requirements

- Visual Studio 2022 17.8+ with .NET MAUI workload, OR VS Code + MAUI extension
- .NET SDK 10.0+
- For iOS deployment: macOS with Xcode 15+ (cross-compile from Windows with Mac Build Host)
- For Android: Android SDK 21+ (API level 5.0 Lollipop minimum)

## Build

```bash
# Android (emulator or device)
dotnet build Optimizer.Mobile/Optimizer.Mobile.csproj -f net10.0-android

# iOS (requires Mac Build Host or macOS)
dotnet build Optimizer.Mobile/Optimizer.Mobile.csproj -f net10.0-ios
```

## Run

```bash
# Android emulator
dotnet build -t:Run -f net10.0-android -p:AndroidLinkMode=None

# iOS simulator (Mac only)
dotnet build -t:Run -f net10.0-ios
```

## Connection

On first launch the setup screen asks for:

- **Server URL** — e.g. `http://192.168.1.10:8765` (find your desktop IP in Settings > Network)
- **API Token** — copy from Optimizer desktop > Settings > Remote API > Copy Token

Both phone and desktop must be on the same Wi-Fi network.

## Features

| Tab | Description |
|-----|-------------|
| Status | Live CPU/Memory/GPU usage with progress bars; temperature and power sensors; Quick Cleanup button |
| Profiles | List and apply all optimization profiles defined in the desktop app |
| Tips | System recommendations generated by the desktop recommendations engine |
| Settings | Connection info, test connectivity, disconnect |

## Project structure

```
Optimizer.Mobile/
  Pages/          — XAML content pages (one per tab + setup)
  ViewModels/     — CommunityToolkit.Mvvm observable ViewModels
  Services/       — ApiClient (HTTP wrapper around the desktop REST API)
  Platforms/
    Android/      — MainActivity, MainApplication, AndroidManifest.xml
    iOS/          — AppDelegate, Info.plist, Program.cs
  Resources/
    Fonts/        — Bundled fonts (add OpenSans-Regular.ttf / OpenSans-Semibold.ttf here)
    Images/       — App icon SVGs
```

## iOS local network permission

`Info.plist` already includes `NSLocalNetworkUsageDescription` and `NSAllowsLocalNetworking` so the app can reach the desktop over HTTP on the LAN without App Transport Security blocking it.
